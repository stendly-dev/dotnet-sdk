# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security of Stendly SDKs seriously. If you believe you've found a security vulnerability, please report it by emailing security@stendly.com.

Please do NOT report security vulnerabilities through public GitHub issues.

**Include in your report:**
- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

## Preferred Languages

We prefer all communications to be in English.

## Policy

- We will acknowledge receipt of your vulnerability report within 48 hours
- We will send a more detailed response within 72 hours
- We will notify you when the vulnerability is fixed
- We will publicly disclose the vulnerability after the fix

## Scope

The following are considered in-scope:
- Stendly SDK source code
- Stendly API endpoints
- Stendly webhook signature verification

## Out of Scope

- Third-party dependencies
- Issues requiring physical access to a user's device
- Social engineering attacks
- Denial of service attacks

## Safe Harbor

We will not pursue legal action against researchers who:
- Make a good faith effort to avoid privacy violations
- Do not cause harm to users or systems
- Do not access or modify user data without permission
- Report vulnerabilities promptly
- Do not publicly disclose before a fix is available

**Thank you for helping keep Stendly and our users safe!**
